# Agent Instructions

## Project overview

Static meme site for the Julio Iglesias July meme tradition. Hosted on GitHub Pages with Cloudflare proxy. No framework, no dependencies — vanilla HTML/CSS/JS.

- **Domain**: memesdejulio.com.ar
- **Repo**: FedericoGarcia/memes-de-julio
- **Stack**: HTML, CSS, JS (vanilla), GitHub Pages, Cloudflare

## Architecture

```
src/                        # Source files (readable)
├── index.html              # Main page with SEO meta tags
├── 404.html                # Custom 404 page
├── script.js               # All client-side logic (IIFE)
├── style.css               # Dark theme, responsive, portrait/landscape
├── catalogs/               # JSON catalogs (auto-included in build via glob)
│   ├── memes.json          # July memes catalog (array with date ranges)
│   ├── countdown.json      # Countdown images catalog (minDays/maxDays)
│   ├── specials.json       # Special date easter eggs (month/from/to)
│   └── generic.json        # Generic memes shown in any state (no date constraints)
├── images/                 # All images (WebP preferred)
│   ├── countdown/          # Countdown-specific images
│   ├── specials/           # Special date images
│   └── inbox/              # (gitignored) staging area for new memes
└── (robots.txt, sitemap.xml, llms.txt, og-image.png, favicon.svg, site.webmanifest)

_site/                      # Build output (gitignored, generated by CI)
scripts/                    # Utility scripts
├── convert-to-webp.sh      # Batch convert images to WebP
└── validate-catalogs.py    # Validate catalog IDs and image files
.github/
├── workflows/deploy.yml    # CI: minify → deploy → lighthouse
├── lighthouserc.json       # Lighthouse assertions
└── dependabot.yml          # Auto-update GitHub Actions
CNAME                       # GitHub Pages custom domain
```

## Design

- **Theme**: "Julio Gold" — warm brown background (`#1c1010`), gold text (`#fef3c7`), amber accent (`#f59e0b`)
- **Heading font**: Lobster (Google Fonts)
- **Body font**: system font stack
- **Suggest modal**: footer button opens a modal showing `memes-con-nivel` image + email link

## State priority

The script resolves which view to show in this order:

1. `?id=` + `from=galeria` → gallery detail view with back button
2. `?id=` query param → show specific meme (searches all catalogs)
3. `?view=galeria` → gallery grid with all memes from all catalogs
4. Special date match (`specials.json`) → show special image
5. July (month === 6) → random meme for the day from `memes.json`
6. Rest of year → countdown with random image from `countdown.json`

## Catalog schemas

### memes.json

```json
{ "id": "slug", "file": "image.webp", "alt": "...", "from": 1, "to": 31, "weekday": 2 }
```

- `from`/`to`: inclusive day range (1-31)
- `weekday` (optional): only show when day `from` falls on this weekday (0=Sun, 6=Sat)

### countdown.json

```json
{ "id": "slug", "file": "countdown/image.webp", "alt": "...", "minDays": 1, "maxDays": 90 }
```

### specials.json

```json
{ "id": "slug", "file": "specials/image.webp", "alt": "...", "caption": "...", "month": 12, "from": 24, "to": 25 }
```

## Naming conventions

- **Files**: kebab-case (`julio-entero.webp`, `feliz-julio-nuevo.webp`)
- **IDs**: same as filename without extension (`julio-entero`, `feliz-julio-nuevo`)
- **Alt text**: describe the image content and any text overlay for SEO and accessibility
- **Images**: WebP format, use `scripts/convert-to-webp.sh` to convert

## Build and deploy

On push to `main`:
1. **Build**: terser (JS), csso (CSS), python (JSON minification)
2. **Deploy**: GitHub Pages via Actions
3. **Lighthouse**: audit against live URL (performance ≥85, rest ≥90)

GitHub Actions are pinned to commit SHAs. Dependabot keeps them updated.

## Adding content

### New meme

1. Add WebP image to `src/images/`
2. Add entry to `src/catalogs/memes.json` with id, file, alt, from, to
3. Push to `main`

### New countdown image

1. Add WebP image to `src/images/countdown/`
2. Add entry to `src/catalogs/countdown.json`

### New special date

1. Add WebP image to `src/images/specials/`
2. Add entry to `src/catalogs/specials.json`

### New generic meme

1. Add WebP image to `src/images/`
2. Add entry to `src/catalogs/generic.json` with id, file, alt
3. Push to `main`

### Image workflow

1. Drop raw images in `src/images/inbox/` (gitignored)
2. Run `/scan-memes` to analyze, rename, and triage (or manually: `./scripts/convert-to-webp.sh`)
3. Review the triage summary, then run `/add-meme <path>` for each approved image
4. For batch processing, use the `meme-processor` agent

For manual workflow without skills:
1. Run `./scripts/convert-to-webp.sh` to convert to WebP
2. Move converted files to proper directory
3. Update the corresponding JSON catalog
4. Delete inbox originals

## Automation

### Skills

- `/scan-memes [path]` — Triage inbox images: analyze, rename, detect duplicates, generate summary. Default path: `src/images/inbox/`
- `/add-meme <image-path>` — Add a single meme: analyze, convert to WebP, update JSON catalog

### Agents

- `meme-processor` — Batch process memes from inbox. Use whenever there are multiple images to process. Two phases: scan (generates `triage-report.json`) then load (processes approved items). Runs in background with worktree isolation.

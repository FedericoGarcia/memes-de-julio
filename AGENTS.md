# Agent Instructions

## Project overview

Static meme site for the Julio Iglesias July meme tradition. Hosted on GitHub Pages with Cloudflare proxy. No framework, no dependencies — vanilla HTML/CSS/JS.

- **Domain**: memesdejulio.com.ar
- **Repo**: FedericoGarcia/memes-de-julio
- **Stack**: HTML, CSS, JS (vanilla), GitHub Pages, Cloudflare

## Architecture

```
src/                        # Source files (readable)
├── index.html              # Main page with SEO meta tags
├── 404.html                # Custom 404 page
├── script.js               # All client-side logic (IIFE)
├── style.css               # Dark theme, responsive, portrait/landscape
├── memes.json              # July memes catalog (array with date ranges)
├── countdown.json          # Countdown images catalog (minDays/maxDays)
├── specials.json            # Special date easter eggs (month/from/to)
├── generic.json            # Generic memes shown in any state (no date constraints)
├── images/                 # All images (WebP preferred)
│   ├── countdown/          # Countdown-specific images
│   ├── specials/           # Special date images
│   └── inbox/              # (gitignored) staging area for new memes
└── (robots.txt, sitemap.xml, llms.txt, og-image.png, favicon.svg, site.webmanifest)

_site/                      # Build output (gitignored, generated by CI)
scripts/                    # Utility scripts (convert-to-webp.sh)
.github/
├── workflows/deploy.yml    # CI: minify → deploy → lighthouse
├── lighthouserc.json       # Lighthouse assertions
└── dependabot.yml          # Auto-update GitHub Actions
CNAME                       # GitHub Pages custom domain
```

## State priority

The script resolves which view to show in this order:

1. `?id=` query param → show specific meme (searches all 3 catalogs)
2. `?view=galeria` → gallery grid with all memes from all catalogs
3. Special date match (`specials.json`) → show special image
4. July (month === 6) → random meme for the day from `memes.json`
5. August–December → "Nos vemos el año que viene"
6. January–June → countdown with random image from `countdown.json`

## Catalog schemas

### memes.json

```json
{ "id": "slug", "file": "image.webp", "alt": "...", "from": 1, "to": 31, "weekday": 2 }
```

- `from`/`to`: inclusive day range (1-31)
- `weekday` (optional): only show when day `from` falls on this weekday (0=Sun, 6=Sat)

### countdown.json

```json
{ "id": "slug", "file": "countdown/image.webp", "alt": "...", "minDays": 1, "maxDays": 90 }
```

### specials.json

```json
{ "id": "slug", "file": "specials/image.webp", "alt": "...", "caption": "...", "month": 12, "from": 24, "to": 25 }
```

## Naming conventions

- **Files**: kebab-case (`julio-entero.webp`, `feliz-julio-nuevo.webp`)
- **IDs**: same as filename without extension (`julio-entero`, `feliz-julio-nuevo`)
- **Alt text**: describe the image content and any text overlay for SEO and accessibility
- **Images**: WebP format, use `scripts/convert-to-webp.sh` to convert

## Build and deploy

On push to `main`:
1. **Build**: terser (JS), csso (CSS), python (JSON minification)
2. **Deploy**: GitHub Pages via Actions
3. **Lighthouse**: audit against live URL (performance ≥85, rest ≥90)

GitHub Actions are pinned to commit SHAs. Dependabot keeps them updated.

## Adding content

### New meme

1. Add WebP image to `src/images/`
2. Add entry to `src/memes.json` with id, file, alt, from, to
3. Push to `main`

### New countdown image

1. Add WebP image to `src/images/countdown/`
2. Add entry to `src/countdown.json`

### New special date

1. Add WebP image to `src/images/specials/`
2. Add entry to `src/specials.json`

### New generic meme

1. Add WebP image to `src/images/`
2. Add entry to `src/generic.json` with id, file, alt
3. Push to `main`

### Image workflow

1. Drop raw images in `src/images/inbox/` (gitignored)
2. Run `./scripts/convert-to-webp.sh` to convert to WebP
3. Move converted files to proper directory
4. Update the corresponding JSON catalog
5. Delete inbox originals

## Infrastructure

- **DNS**: Cloudflare (proxy enabled, Managed robots.txt OFF)
- **Email**: Cloudflare Email Routing → `memes@memesdejulio.com.ar` → `memesdejulio.com.ar@gmail.com`
- **SSL**: Full (Strict) via Cloudflare + GitHub Pages Let's Encrypt cert
- **Analytics**: Cloudflare Web Analytics (beacon.min.js, allowed in CSP)
